# mapp-aiyouxi-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定域名或具体网站，仅作为通用资源存储与分发使用。

## 项目简介

mapp-aiyouxi-resource 是一个静态资源仓库，主要存放各类独立 HTML 页面。每个页面均为自包含的 HTML 文件，可直接在浏览器中打开运行，无需额外服务器环境或依赖。

本仓库适用于个人或团队存放、共享以及版本管理 HTML 页面资源，方便快速获取历史版本或进行协作维护。

## 目录说明

```
mapp-aiyouxi-resource/
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 页面引用的公共资源（如样式、脚本、图片等）
├── docs/           # 相关文档或说明
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件代表一个独立的 HTML 页面项目。
- `assets/` 目录用于存放多个页面可能共用的资源文件。
- `docs/` 目录可放置使用说明、变更记录或其他辅助文档。

## 页面归档说明

仓库中的 HTML 页面按照以下原则进行归档：

1. **独立性**：每个页面文件应尽可能自包含，减少对外部资源的依赖。
2. **版本化**：通过 Git 提交记录追踪每次变更，方便回溯与比较。
3. **命名规范**：页面文件或文件夹名称应简洁明了，反映其内容或用途。
4. **无敏感信息**：页面中不得包含任何个人隐私、密钥、敏感数据或指向特定域名的链接。

## 使用方式

- 直接下载仓库中的 HTML 文件，在浏览器中打开即可使用。
- 可通过 GitHub Pages 或其他静态托管服务部署 `pages/` 目录下的页面。
- 如需引用仓库中的资源，请使用相对路径或通过 GitHub 原始文件链接获取。

## 维护说明

### 提交准则

- 每次提交应只涉及一个页面或一组相关变更。
- 提交信息应简要说明变更内容，便于后续查阅。
- 避免提交与页面无关的大型文件（如二进制文件、编译产物等）。

### 分支管理

- `main` 分支用于存放稳定版本。
- 可创建 `dev` 或 `feature-*` 分支进行开发与测试，待验证无误后合并至 `main`。

### 问题与建议

如发现页面错误或有改进建议，欢迎提交 Issue 或 Pull Request。在提交 Issue 时，请尽量描述清晰，包括：
- 页面名称或路径
- 问题表现或建议内容
- 浏览器及操作系统信息（如适用）
